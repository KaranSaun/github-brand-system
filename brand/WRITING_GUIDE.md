# Writing Guide

## Principles

1. **Every word earns its place.** If removing a word doesn't change meaning, remove it.
2. **Be specific.** "Production AI" not "advanced technology."
3. **Show evidence.** Let repositories speak. Don't claim what code doesn't support.
4. **No buzzwords.** "Synergy," "leveraging," "paradigm" — banned.
5. **No exaggeration.** Don't fabricate experience.

## Tone

| Do | Don't |
|---|---|
| "Builds production AI systems" | "Revolutionizing the AI landscape" |
| "FastAPI + Next.js + Docker" | "Cutting-edge tech stack" |
| "Automates warehouse workflows" | "Transforming logistics" |
| "Multi-model AI pipeline" | "Next-gen AI solution" |

## Project Description Format

```
Project Name — One-line description.

[2-3 sentences on problem and solution]

Stack: Python, FastAPI, Next.js, PostgreSQL, Docker
```

## Bio Format

```
[Role] building [what] for [domain].
[Current focus] | Past: [notable work]
```

## Commit Message Format

```
type: short description

- detailed point 1
- detailed point 2
```

Types: `feat`, `fix`, `docs`, `chore`, `refactor`, `style`, `perf`
